# On-call Quickstart Guide


A fast-start guide for engineers handling incidents.


## First 5 Minutes
- [ ] Acknowledge the alert
- [ ] Check dashboards for immediate anomalies
- [ ] Identify scope & impact quickly
- [ ] Communicate initial status in incident channel


## During Investigation
- [ ] Focus on stabilization first
- [ ] Avoid speculative debugging
- [ ] Capture notes/timestamps
- [ ] Escalate if no progress after 15 minutes

## Aftermath
- [ ] Schedule a post-incident review
- [ ] Add metrics, graphs, screenshots to the retro
- [ ] Update runbook if gaps were identified
